Access Modifiers

  • public: visible to all, all other classes and packages may use it
  • private: accessible from the same class only
  • protected: accessible from within the same package
  • default: AKA package-private - all members are visible within the same package, but aren't accessible from other packages

This vs Super

Super is a keyword that is used to access the parent class' variables and methods. This is a keyword that is used to call the current class members' variables and methods and is used when we have a parameter with the same name as a field (instance variable). You can't use these keywords in static areas. You might think of the word super when you think of the OOP concept of Inheritance.

You might also see a call to this or super that looks like: this() or super(). Use this() to call a constructor from another overloaded constructor in the same class. To use this(), it must be the first statement in a constructor. It's used in constructor chaining, when one constructor calls another constructor. When to use Abstract Class versus Interface

Consider using an Abstract class when one thing "is" another: Dog is an Animal, Corvette is a Car. You may want to use an Abstract class for objects that are closely related. Consider using an interface when one thing "has a" or "is capable of" something else: a Parrot CanFly (CanFly is the interface). You may want to use an interface for providing common functionality among unrelated classes.

In an interface, all methods are automatically public, all variables in the interface (if any) must be public static final (they have to be static because you can't instantiate an interface) and all methods in an interface are abstract.

When you have an Abstract Class, the subclass must provide implementations for all of the abstract methods in its parent class. If it doesn't, then the subclass must also be declared abstract.
